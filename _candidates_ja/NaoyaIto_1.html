Elixir は Erlang VM の上で動作する動的型付けの関数型言語です。 おや、Scala が Java VM の上で動作する〜、というのにそっくりですね。

Elixir は Erlang/OTP の特徴をそのまま受け継いでおり、その中にはパターンマッチやアクターモデルによる並行処理など Scala プログラマにもお馴染みの機能/モデルが多数含まれます。

一方、静的型付けの Scala とは異なり動的型付けであるがゆえに異なる特徴を持つ部分もあります。たとえばパターンマッチのユースケース一つとっても、動的であるがゆえに、言語デザイン的にはより広範囲への応用が見られます。

Elixir は 2015 年昨今 Web 開発の Next big language だ、などと騒ぎになっています。果たしてその実力は?

そんな Elixir の特徴を、Scala と比較しながら Scala プログラマの皆さんに紹介し、その反応を見てみたいと思います。本格利用はもちろん、Scala 好きなみなさんが「ちょっとカジュアルに関数型したいな」とか「カジュアルにアクターしたいな」というときに使える言語としても興味を持っていただけるのではないか、と思います。
